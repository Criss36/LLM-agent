<task type="auto">
  <name>Task 1: 创建 GitHub Actions CI/CD 工作流</name>
  <files>
    .github/workflows/deploy.yml
  </files>
  <action>
    创建 `.github/workflows/deploy.yml`，包含以下结构:

    **触发器:**
    - push 到 main 分支
    - pull_request 到 main 分支

    **并发控制:**
    - concurrency 分组为 pages，cancel-in-progress: false 防止并发部署冲突

    **build 作业:**
    - runs-on: ubuntu-latest
    - permissions: contents: read（最小权限原则）
    - 步骤:
      1. actions/checkout@v4
      2. actions/setup-node@v4, node-version: 20, cache: npm
      3. npm ci（使用 package-lock.json，比 npm install 更快更确定）
      4. npm run lint（加 continue-on-error: true，lint 不阻断部署但日志可见）
      5. npm run test:run（测试失败 → 阻断流水线）
      6. npm run build（构建失败 → 阻断流水线）
      7. actions/upload-pages-artifact@v3, path: dist/（上传构建产物）

    **deploy 作业:**
    - if: github.ref == 'refs/heads/main'（仅 main 分支触发部署）
    - needs: build（等待 build 成功）
    - runs-on: ubuntu-latest
    - permissions: pages: write, id-token: write
    - environment: name: github-pages, url: ${{ steps.deployment.outputs.page_url }}
    - 步骤:
      1. actions/deploy-pages@v4

    **约束:**
    - 只使用 actions/ 命名空间下的官方 action（checkout, setup-node, upload-pages-artifact, deploy-pages）
    - 不使用第三方 action，降低供应链风险
    - 不绑定任何自定义 secret（GITHUB_TOKEN 自动注入）
  </action>
  <verify>
    <automated>node -e "const fs=require('fs');require('js-yaml').load(fs.readFileSync('.github/workflows/deploy.yml','utf8'))"</automated>
  </verify>
  <done>
    - .github/workflows/deploy.yml 文件存在
    - YAML 语法解析通过，无错误
    - 文件包含 on.push.branches.main 触发器
    - 文件包含 build 和 deploy 两个作业
    - build 作业包含 test 步骤
    - deploy 作业使用 deploy-pages action 且有限定为 main 分支的条件
  </done>

<task type="auto">
  <name>Task 2: 配置 Vite base 路径以适配 GitHub Pages 子路径部署</name>
  <files>
    vite.config.ts
  </files>
  <action>
    修改 `vite.config.ts`，在 defineConfig 调用中添加 base 字段:

    方案: 使用环境变量 GITHUB_ACTIONS（GitHub Actions 内置变量，始终为 true）做条件判断:
    - CI 环境: base: '/LLM-agent/'
    - 本地开发: base: '/'

    修改后的 vite.config.ts 结构:
    ```ts
    export default defineConfig({
      base: process.env.GITHUB_ACTIONS ? '/LLM-agent/' : '/',
      plugins: [react(), tailwindcss()],
      test: {
        environment: 'jsdom',
        globals: true,
        setupFiles: './src/test/setup.ts',
      },
    })
    ```

    原因: 仓库名为 Criss36/LLM-agent，GitHub Pages 部署到 https://criss36.github.io/LLM-agent/。如果 base 保持默认 /，Vite 构建出的 HTML 中脚本路径为 "/assets/index-xxx.js"，浏览器会请求 https://criss36.github.io/assets/（404）。设置 base: '/LLM-agent/' 后路径变为 "/LLM-agent/assets/index-xxx.js"，正确指向 Pages 子路径。

    注意: 只添加 base 字段一行，保留 plugin 和 test 配置原样不动。不要修改其他任何内容。
  </action>
  <verify>
    <automated>grep -q "base:" vite.config.ts</automated>
  </verify>
  <done>
    - vite.config.ts 包含 base 字段
    - base 值有条件判断 /LLM-agent/（CI）和 /（本地）
    - 原有 plugin 和 test 配置保持不变
    - npm run build 在本地环境仍可正常工作（base="/"）
  </done>
</task>
