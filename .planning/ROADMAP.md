# 路线图: Criss36 LLM 工程师个人主页

## 概述

这是一个存量打磨项目。站点已有完整的功能体系（导航、英雄区、项目网格、博客、交互式 Demo、技术文档），当前阶段的目标是消除已知的稳定性问题、完善内容、增强无障碍性、优化性能，并建立测试和 CI/CD 基础设施。每一次交付都应让访客感受到更流畅、更完整、更专业的体验。

## 阶段

- [ ] **阶段 1: 高 ROI 缺陷修复与样式清理** - 修复内存泄漏、动画重触发 bug，清理重复 CSS
- [ ] **阶段 2: 内容完善与数据更新** - 补充缺失技术文档，更新作品集数据源
- [ ] **阶段 3: 无障碍与性能优化** - 键盘导航、屏幕阅读器支持、图片懒加载、字体子集化
- [ ] **阶段 4: 测试基础设施** - 建立 Vitest 单元测试和 Playwright E2E 测试
- [ ] **阶段 5: CI/CD 流水线** - GitHub Actions 自动构建 + 部署到 GitHub Pages

## 阶段详情

### 阶段 1: 高 ROI 缺陷修复与样式清理
**目标**: 站点稳定运行，无内存泄漏、动画卡顿或 CSS 样式冲突
**依赖**: 无（首个阶段）
**需求**: PERF-01, PERF-02, STYLE-01
**成功标准**（必须为真的行为）:
  1. 进入英雄区观看打字机动画，导航离开后重新进入 —— 动画正确重启，不会卡死在中间状态
  2. 在页面各区域间快速切换导航多次 —— 控制台无 `setState on unmounted component` 警告，无内存持续增长
  3. 使用 Tab 键遍历所有可交互元素 —— `:focus-visible` 环形指示器每次只出现一次，样式一致无重复
**计划**: TBD
**UI 提示**: 是

### 阶段 2: 内容完善与数据更新
**目标**: 作品集展示最前沿的项目经历、技能栈和技术深度内容
**依赖**: 阶段 1
**需求**: CONTENT-01, CONTENT-02
**成功标准**（必须为真的行为）:
  1. 访问 docs/ 部分并点击每篇技术文档链接 —— 全部跳转到实际内容，无 404 或占位符页面
  2. 查看项目展示区、技能网格和博客列表 —— 所有数据为最新信息，无过时内容
**计划**: TBD

### 阶段 3: 无障碍与性能优化
**目标**: 站点可通过键盘完整操作，各种网络条件下加载高效
**依赖**: 阶段 2
**需求**: A11Y-01, PERF-03
**成功标准**（必须为真的行为）:
  1. 仅使用键盘（Tab/Enter/Escape/方向键）完成整个站点的导航和交互 —— 所有菜单、链接、按钮均可到达和操作
  2. 屏幕阅读器（NVDA/VoiceOver）正确朗读所有有意义的内容，角色和标签准确无误
  3. 打开 DevTools Network 面板并刷新 —— 首屏以下图片懒加载加载，字体为子集化版本，JS 打包体积较优化前明显减小
**计划**: TBD
**UI 提示**: 是

### 阶段 4: 测试基础设施
**目标**: 核心组件拥有自动化测试，每次运行可靠通过
**依赖**: 阶段 3
**需求**: TEST-01
**成功标准**（必须为真的行为）:
  1. 运行 `npm run test` —— 所有 Vitest 单元测试通过，零失败
  2. 运行 `npm run test:e2e` —— 所有 Playwright E2E 测试覆盖关键用户流程并通过
  3. 修改一个核心组件文件 —— 重新运行测试时，相关快照测试检测到变更并提示更新
**计划**: TBD

### 阶段 5: CI/CD 流水线
**目标**: 每次推送到主分支都触发自动构建和部署
**依赖**: 阶段 4
**需求**: CI-01
**成功标准**（必须为真的行为）:
  1. 推送提交到 main 分支 —— GitHub Actions 工作流自动触发
  2. 访问仓库 Actions 标签页 —— 构建步骤通过，部署步骤成功
  3. 访问生产环境 URL（GitHub Pages） —— 最新变更在几分钟内上线
  4. 查看 Actions 日志 —— 测试套件作为流水线一部分运行并通过
